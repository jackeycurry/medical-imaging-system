# 胸片检测模块设计方案

## 背景

医疗管理系统新增胸片检测功能，作为独立分栏。医生在诊疗时可通过弹窗上传胸片图片，调用后端胸片分析服务（8000端口），查看原图、标注图、检测结果、AI报告，并选择性存入数据库关联患者。

## 需求汇总

| 项目 | 决策 |
|------|------|
| 使用场景 | 医生诊疗时实时分析 |
| 患者关联 | 可选择已有患者或直接输入新建 |
| 结果展示 | 详细检测框标注图 + 各异常置信度列表 + PDF报告 |
| PDF存储 | 存文件路径，数据库存路径 |
| 交互方式 | 弹窗流程 |
| 错误处理 | 友好错误提示 |

## 数据模型

### MedicalImage 表（新增字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| report_id | String(100) | 报告ID，对应PDF文件名 |
| pdf_path | String(500) | PDF文件存储路径 |

## API 设计

### 1. 分析接口

**POST /api/chest-xray/analyze**

请求：FormData
- file: 图片文件
- user_info: JSON字符串 {name, age, gender}（可选）

响应：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "report_id": "abc123",
    "detections": [...],
    "ai_report": {...},
    "original_image": "base64...",
    "annotated_image": "base64...",
    "pdf_url": "/api/report/abc123/pdf"
  }
}
```

### 2. 保存接口

**POST /api/chest-xray/save**

请求：
```json
{
  "patient_name": "张三",
  "patient_id": "P001",
  "report_id": "abc123",
  "detections": "[...]",
  "report_content": "AI报告内容",
  "pdf_path": "/backend/reports/abc123.pdf"
}
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 8000端口服务不可用 | 返回"胸片分析服务暂不可用，请稍后重试" |
| 图片格式不支持 | 返回"不支持的图片格式" |
| 分析超时 | 返回"分析超时，请重试" |
| 保存失败 | 提示保存失败，可重试 |
